Get Help & Development
======================

From questions to citations to contributions, this page gathers the links you need to collaborate with the Coord2Region community.

   :local:
   :depth: 1

Get help
--------

- **Documentation.** Start with the :doc:`documentation_overview`, :doc:`pipeline`, and :doc:`tutorials` pages for self-serve answers.
- **Issues.** Report bugs or request features on `GitHub Issues <https://github.com/babasanfour/Coord2Region/issues>`_. Please include CLI/Python versions, operating system, and snippets from ``coord2region-output/`` when possible.
- **Security.** Follow :doc:`SECURITY` for responsible disclosure guidelines.
- **Conduct.** The :doc:`CODE_OF_CONDUCT` applies to every interaction—GitHub, chat, conferences, and publications.

.. _cite-coord2region:

Cite Coord2Region
-----------------

Please cite Coord2Region (and any atlases, models, or providers you rely on) when publishing or presenting your work.

- Preferred citation (CFF format) lives in :download:`CITATION.cff <../../CITATION.cff>`.
- For BibTeX/APA conversions, paste the CFF file into `cffconvert <https://github.com/citation-file-format/cff-converter-python>`_ or use GitHub's "Cite this repository" button.
- Mention software version numbers (``coord2region --version``) plus the atlases/providers enumerated in your YAML/JSON output for reproducibility.
